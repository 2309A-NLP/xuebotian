# 小家记账本智能体 - 设计方案

## 1. 项目概述

### 1.1 项目背景
家庭记账是日常财务管理的基础工作，传统方式需要手动记录或使用固定模板的记账软件。本项目旨在利用大语言模型(LLM)的自然语言理解能力，实现"说话就能记账"的智能记账体验。

### 1.2 项目目标
- 用户通过自然语言描述即可完成记账、查询、统计、删除等操作
- 无需学习SQL或复杂操作，降低使用门槛
- 核心SQL由AI自动生成，保证灵活性的同时确保安全

### 1.3 核心价值
| 价值点 | 说明 |
|--------|------|
| 零学习成本 | 说话就能记账，不需要学习任何技术 |
| 高度灵活 | 任何自然语言表达都能理解 |
| 安全可控 | 严格限制SQL操作类型，防止数据泄露 |

## 3. 数据库设计

### 3.1 表结构

```sql
CREATE TABLE money_notes (
    id INTEGER PRIMARY KEY AUTOINCREMENT,  -- 自增主键
    date TEXT NOT NULL,                      -- 日期(YYYY-MM-DD)
    member TEXT NOT NULL,                    -- 成员(爸爸/妈妈/女儿)
    category TEXT NOT NULL,                  -- 类别
    item TEXT NOT NULL,                      -- 物品/事项名称
    amount REAL NOT NULL,                    -- 金额
    type TEXT NOT NULL,                      -- 类型(收入/支出)
    note TEXT,                               -- 备注
    created_at TEXT DEFAULT CURRENT_TIMESTAMP -- 创建时间
);
```

### 3.2 索引设计

```sql
CREATE INDEX idx_member ON money_notes(member);  -- 按成员查询优化
CREATE INDEX idx_date ON money_notes(date);     -- 按日期查询优化
CREATE INDEX idx_type ON money_notes(type);      -- 按类型查询优化
```

### 3.3 数据字典

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| id | INTEGER | 记录唯一标识 | 1, 2, 3... |
| date | TEXT | 记账日期 | 2026-06-23 |
| member | TEXT | 家庭成员 | 爸爸、妈妈、女儿 |
| category | TEXT | 消费类别 | 购物、餐饮、旅游 |
| item | TEXT | 具体物品 | 登山鞋、午餐费 |
| amount | REAL | 金额(元) | 499.00 |
| type | TEXT | 收支类型 | 收入、支出 |
| note | TEXT | 备注信息 | 可选 |
| created_at | TEXT | 创建时间戳 | 2026-06-23 19:00 |

### 3.4 成员与类别枚举

```
家庭成员: 爸爸、妈妈、女儿

支出类别: 餐饮、交通、购物、娱乐、教育、医疗、旅游、服装、日用品、通讯、居住、书籍、其他

收入类别: 工资、奖金、报销、投资收益、兼职、补贴、其他
```

## 4. 核心模块设计

### 4.1 SQL生成器 (generate_sql)

**设计思路**: 通过精心设计的Prompt，让LLM理解表结构、规则后自动生成SQL。

**Prompt结构**:
```
角色设定: SQL生成专家
当前时间: {动态获取}
表结构: CREATE TABLE ... (附带注释说明)
成员列表: 爸爸、妈妈、女儿
类别列表: 支出类别 + 收入类别
时间规则: 今天/昨天/本月 等自然语言的日期映射
类型规则:买东西=支出、收到=收入
金额规则: 提取数字
类别映射: 买书→书籍、旅游→旅游
输出要求: 只返回SQL语句
```

**输出示例**:
```
输入: "今天女儿买了双登山鞋499元"
输出: INSERT INTO money_notes (date, member, category, item, amount, type)
      VALUES ('2026-06-23', '女儿', '购物', '登山鞋', 499, '支出')
```

### 4.2 SQL校验器 (_validate_sql)

**设计思路**: 白名单机制，只允许预定义的安全操作。

**校验规则**:
| 检查项 | 规则 |
|--------|------|
| SQL首词 | 必须在允许列表中 |
| 允许列表 | SELECT、INSERT、UPDATE、DELETE |
| 禁止列表 | DROP、ALTER、CREATE、TRUNCATE、EXEC |

**安全考量**:
- 只检查SQL首词，不检查内容（如字段名包含CREATE不影响）
- 禁止所有DDL和存储过程操作
- 不允许批量操作（如UNION合并查询）

### 4.3 结果格式化器 (_format_result)

**设计思路**: 二次调用LLM，将SQL执行结果(JSON)转换为人类友好的自然语言回复。

**Prompt结构**:
```
原始用户输入: {user_input}
执行的SQL: {sql}
执行结果: {json_result}

回复要求:
1. 简洁友好
2. 涉及金额说明单位(元)
3. 查询结果要清晰列出
4. 如果是添加成功，说明记录了什么
5. 如果是删除成功，说明删除了什么
6. 如果没有数据，说明情况
```

## 6. 安全设计

### 6.1 多层防护

```
┌─────────────────────────────────────────┐
│  第1层: API密钥验证                      │
│  - 程序启动时验证API Key有效性           │
│  - 无Key时进入模拟模式(只读演示)         │
└─────────────────────────────────────────┘
           ↓
┌─────────────────────────────────────────┐
│  第2层: SQL白名单校验                    │
│  - 只允许 SELECT/INSERT/UPDATE/DELETE   │
│  - 禁止 DROP/ALTER/CREATE 等危险操作    │
└─────────────────────────────────────────┘
           ↓
┌─────────────────────────────────────────┐
│  第3层: SQLite权限控制                   │
│  - 文件级权限控制(操作系统层)            │
│  - 建议数据库文件600权限                 │
└─────────────────────────────────────────┘
```

### 6.2 防护效果

| 攻击类型 | 是否防御 | 防御方式 |
|----------|----------|----------|
| SQL注入 | ✅ | 白名单校验 |
| 数据删除 | ❌ | 允许DELETE(业务需求) |
| 数据篡改 | ✅ | 需要精确描述 |
| 批量查询 | ✅ | 不支持UNION |
| 越权访问 | ✅ | 只能操作单表 |

## 7. 错误处理

### 7.1 错误分类

| 错误类型 | 触发条件 | 用户提示 |
|----------|----------|----------|
| 无API Key | 未设置API Key | 进入模拟模式 |
| LLM调用失败 | API不可用/限流 | "API调用失败，请检查网络" |
| SQL校验失败 | 生成禁止的SQL | "禁止执行此操作" |
| SQL执行失败 | SQL语法错误 | 显示错误信息+生成的SQL |
| 数据库错误 | 文件损坏/权限 | "数据库错误，请检查" |

### 7.2 异常处理流程

```
用户输入
    ↓
生成SQL ──── 失败 ───→ 返回"SQL生成失败"
    ↓
校验SQL ──── 失败 ───→ 返回"禁止执行此操作"
    ↓
执行SQL ──── 失败 ───→ 返回"SQL执行失败"+错误信息
    ↓
格式化结果 ── 失败 ──→ 返回原始JSON结果
    ↓
返回用户
```

## 8. 技术选型

| 组件 | 选型 | 理由 |
|------|------|------|
| 编程语言 | Python 3.10+ | 生态丰富，LLM集成友好 |

| LLM模型 | Qwen2.5-72B | 阿里开源，效果好，性价比高 |
| LLM服务商 | 硅基流动 | 国内可用，OpenAI兼容 |
| 数据库 | SQLite | 零配置，嵌入式，适合单机 |
| 部署方式 | 本地运行 | 家庭场景，无需服务器 |
