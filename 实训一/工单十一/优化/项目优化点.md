# RAG 工单项目已完成优化点

## 1. PDF 解析与中文文本处理优化

项目使用 `unstructured.partition_pdf` 对 PDF 进行结构化解析，并设置 `languages=["chi_sim"]` 以适配中文文档内容。相比直接读取 PDF 原始文本，这种方式能更好地区分标题、正文、表格等文档元素，并通过 `Counter(type(element) for element in elements)` 统计解析结果，便于观察 PDF 内容抽取质量。

## 2. 文本切块策略优化

在文本切分阶段，项目没有简单按固定字符数硬切，而是先使用 NLTK 进行句子级切分，再按 `chunk_size=300` 组合文本块，并设置 `overlap=45` 保留相邻文本块之间的上下文。这样可以降低语义被截断的概率，让后续生成的 QA 数据和 Embedding 训练样本更连贯。

## 3. QA 数据生成质量控制

项目通过提示词约束大模型按固定格式输出：

```text
Question: ...
Answer: ...
```

程序再根据 `Answer:` 对返回内容进行解析，并使用 `is_valid_text` 过滤空字符串、无效值和 `N/A`。这一步减少了脏数据进入训练集的风险，使 `generated_qa_pairs.csv` 更适合作为微调数据源。

## 4. Embedding 微调方法优化

训练脚本采用 `MultipleNegativesRankingLoss`，把问题作为 `anchor`，对应文本块作为 `positive`，适合优化语义检索场景中的问题到文档片段匹配能力。同时叠加 `MatryoshkaLoss`，在 768、512、256、128、64 多个向量维度上共同训练，使模型既能保留高维检索效果，也能支持低维压缩部署。

## 5. 训练配置与资源利用优化

训练阶段使用了以下配置来平衡效果和显存占用：

- `per_device_train_batch_size=4`
- `gradient_accumulation_steps=16`
- `learning_rate=2e-5`
- `lr_scheduler_type="cosine"`
- `bf16=True`
- `tf32=True`
- `batch_sampler=BatchSamplers.NO_DUPLICATES`

其中梯度累积可以在小 batch 下模拟更大的有效 batch，`NO_DUPLICATES` 有助于 Multiple Negatives 训练时减少重复样本带来的干扰。

## 6. 多维度检索评估优化

项目为 768、512、256、128、64 五个 Matryoshka 维度分别构建 `InformationRetrievalEvaluator`，统一使用 cosine 相似度和 `nDCG@10` 评估检索排序效果。这样不仅能判断微调是否提升了模型，还能比较不同向量维度下的效果与资源成本。

## 7. 训练前后效果对比

验证集结果显示，微调后所有维度的 `nDCG@10` 均有提升：

| 维度 | 训练前 nDCG@10 | 训练后 nDCG@10 | 提升 |
| ---: | ---: | ---: | ---: |
| 768 | 0.8216 | 0.9099 | +0.0883 |
| 512 | 0.7754 | 0.8929 | +0.1175 |
| 256 | 0.7964 | 0.9099 | +0.1135 |
| 128 | 0.7374 | 0.9099 | +0.1725 |
| 64 | 0.6070 | 0.8841 | +0.2771 |

低维度提升尤其明显，说明 Matryoshka 微调增强了压缩向量下的语义检索能力。

## 8. 最终测试效果

最终测试集 `nDCG@10` 结果如下：

| 维度 | 测试集 nDCG@10 |
| ---: | ---: |
| 768 | 0.9333 |
| 512 | 0.9534 |
| 256 | 0.9298 |
| 128 | 0.8274 |
| 64 | 0.7786 |

其中 512 维效果最好，适合追求较高检索质量的场景；128 维虽然效果略低，但可以作为降低向量存储、索引和检索成本的折中方案。

## 9. 后续可继续优化方向

- 将 `SILICONFLOW_API_KEY` 从源码默认值中移除，仅通过环境变量注入。
- 增加 `requirements.txt` 或 `pyproject.toml` 固定依赖版本，提升部署可复现性。
- 将 `data` 目录、模型路径、模型名称、chunk 参数改成命令行参数。
- 扩大验证集和测试集规模，降低当前小样本评估波动。
- 接入向量数据库，补充端到端 RAG 检索与问答评测。
