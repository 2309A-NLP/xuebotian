# RAGAS 测试优化记录

## 1. 优化目标

本轮优化主要围绕两件事：

- 提升召回，避免“明明有证据但没捞到”
- 提升答案贴合度，减少答非所问和证据漂移

## 2. 已做优化

### 2.1 查询侧

- 增加 LLM 查询改写，生成更适合检索的等价问法。
- 对改写结果做安全校验，保留数字、年份、组织名、约束词和证据锚点。
- 多轮对话会被压缩进检索上下文，避免丢失指代信息。

### 2.2 检索侧

- 支持多查询向量检索并合并候选。
- 扩大召回池，再进入 rerank。
- 对不同内容类型做结构化排序加权。
- 表格、图片类证据在最终结果里会尽量保留补位。

### 2.3 排序侧

- 文本块根据标题、章节、段落结构加分。
- 表格块根据 caption、header、行数加分。
- 图片块根据 caption、图表类型、锚点词加分。

### 2.4 Prompt 侧

- 上下文按字符数裁剪，避免 Prompt 过长。
- 对已知表格文本做修复和规范化。
- 多轮对话做截断，保留最近且最相关的内容。

### 2.5 失败回退

- 如果模型输出“证据不足”类回答，会触发原始问题重试。
- 查询改写失败时自动回退到保守版本。
- 检索为空时直接返回空答案，避免幻觉扩写。

## 3. 当前效果

当前结果说明：

- 检索召回已经稳定
- 证据一致性较好
- 答案相关性仍有继续提升空间，主要在更细粒度的问句重写和表格/图片类样本上

## 4. 后续建议

- 扩充表格和图文样本
- 增加多轮对话样本
- 针对低分样本做人工复盘
- 继续优化查询改写的“保守性”和“可检索性”平衡
