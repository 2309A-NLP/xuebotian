对回答速度进行了优化
我这次对速度做的优化，主要是这四类。
第一类是消灭冷启动。
在服务启动时提前做了预热，而不是等用户第一次提问时再加载：
提前加载 BGE-m3
提前跑一次查询向量生成
如果 Milvus 集合存在，提前 load_collection

第二类是减少送给大模型的上下文。
我在 RAG prompt 里把每个召回片段做了裁剪，不再整段都塞给模型，这会直接减少 LLM 的输入长度，从而缩短生成时间。
新增了 PROMPT_CHUNK_CHAR_LIMIT
在组 prompt 时只截前面一部分内容

第三类是把等待感变短。
我把问答改成了 SSE 流式输出，不再等整段答案生成完才返回。这样虽然不一定减少总耗时，但用户能更早看到内容。
后端新增了 /api/chat/stream
LLM 调用改成 stream=True
前端改成边接收边显示

第四类是减少流式卡顿。
流式输出如果每个 token 都立即刷新，视觉上会抖、会卡。我做了两层平滑：
后端把过碎的 token 合并后再推
前端用 requestAnimationFrame 合并 DOM 刷新
