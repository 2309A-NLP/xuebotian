# Qwen3-VL-2B 微调设计文档

## 1. 项目目标

本项目目标是基于 `Qwen3-VL-2B-Instruct` 构建一个面向工业专利图文问答场景的多模态模型，使其在以下能力上明显优于基线模型：

- 专利文本事实抽取
- 图纸标号定位与空间关系判断
- 工业术语识别
- 图文联合推理与结构化回答

从现有评估结果看，微调后模型已经从 `16.67%` 的准确率提升到 `91.67%`，说明当前路线有效，具备继续工程化和部署化的价值。

## 2. 当前方案概览

### 2.1 基础模型

- 基座模型：`Qwen3-VL-2B-Instruct`
- 任务类型：多模态监督微调（SFT）
- 微调方法：LoRA
- 目标场景：专利文本 + 专利附图联合问答

### 2.2 当前训练配置

当前配置来自 `train_qwen3_vl_2b_lora_windows.yaml`：

- `stage: sft`
- `finetuning_type: lora`
- `lora_target: all`
- `lora_rank: 32`
- `lora_alpha: 64`
- `lora_dropout: 0.05`
- `use_rslora: true`
- `dataset: patent_multimodal`
- `cutoff_len: 4096`
- `max_samples: 1000`
- `per_device_train_batch_size: 1`
- `gradient_accumulation_steps: 4`
- `learning_rate: 2e-4`
- `num_train_epochs: 20`
- `lr_scheduler_type: cosine`
- `warmup_steps: 100`
- `bf16: true`
- `val_size: 0.1`
- `eval_strategy: steps`
- `eval_steps: 100`

### 2.3 模型导出配置

当前合并配置来自 `merge_lora_model.yaml`：

- 基座模型：`Qwen3-VL-2B-Instruct`
- Adapter 输出目录：`saves/qwen3-vl-2b-patent-lora`
- 合并后模型目录：`saves/merge_model`
- 导出设备：`cpu`

## 3. 数据设计

### 3.1 数据集形态

当前训练数据在 `data/dataset_info.json` 中注册为 `patent_multimodal`，格式为 `sharegpt` 风格：

- 消息字段：`conversations`
- 图片字段：`images`
- 角色标签：
  - `human`
  - `gpt`
  - `system`

### 3.2 数据特点

当前数据覆盖两类核心任务：

- 专利文本问答
  - 发明人、材料类型、结构特征、尺寸定义、公式含义等
- 专利图示问答
  - 部件标号、相对位置、流动路径、上下游关系、图纸结构解释等

### 3.3 数据风险

当前数据文件 `patent_multimodal_sharegpt_windows.json` 的图片路径是 Windows 绝对路径，例如：

- `G:\...\images\CN100342976C\page1.png`

这意味着原始训练数据不能直接在 Linux 环境复用，部署时必须做路径重映射或重新生成 Linux 版数据文件。

## 4. 设计 rationale

### 4.1 为什么选择 LoRA

- 2B 规模模型适合以 LoRA 方式快速迭代
- 相比全参微调，LoRA 显存压力更低，试错成本更小
- 便于保留基座模型，并将 adapter 与 merge 模型分别管理

### 4.2 为什么选择 `lora_target: all`

当前任务不是单一文本补全，而是图文联合推理。全层 LoRA 比只打注意力层更适合：

- 兼顾视觉理解和语言生成
- 提升跨模态对齐能力
- 更适合复杂结构化回答

### 4.3 为什么使用 RSLoRA

- 在相同 rank 下更稳定
- 对中小模型的训练收敛和泛化通常更友好
- 能降低高学习率下的不稳定风险

### 4.4 为什么 `cutoff_len=4096`

当前输入包含：

- 系统提示
- 问题文本
- 专利背景文本
- 图像占位与多模态上下文

`4096` 能覆盖当前专利问答样本的大部分上下文长度，避免因截断损失关键信息。

## 5. 训练流程设计

建议采用以下标准流水线：

1. 准备数据
   - 校验 `conversations` 字段完整性
   - 校验图片路径可访问性
   - 为 Linux 环境生成路径重映射后的数据文件
2. 执行 LoRA 训练
   - 使用 `llamafactory-cli train`
   - 保留训练日志、loss 曲线和 checkpoint
3. 导出合并模型
   - 使用 `llamafactory-cli export`
   - 生成独立可部署模型目录
4. 训练前后对比评估
   - 使用 `evaluate_model.py`
   - 分别评估基座模型和 merge 模型
5. 产出评估报告
   - 记录总体指标
   - 标记新增答对、仍错误和异常样本

## 6. 当前效果与验收标准

### 6.1 当前效果

根据现有评估结果：

- 训练前准确率：`16.67%`
- 训练后准确率：`91.67%`
- 正确题数：`2/12 -> 11/12`
- 无回退题目
- 唯一残留问题：`CN100347506C_002`

### 6.2 建议验收标准

建议将后续版本验收标准定为：

- 准确率不低于 `90%`
- 工业综合分不低于 `45`
- 图示推理得分不低于 `0.60`
- 不允许出现批量推理异常
- 单条异常样本比例低于 `1%`
